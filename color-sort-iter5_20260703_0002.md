# 色彩排序游戏 MVP 迭代 - 第五轮 (2026-07-03 00:02)

## 目标
继续打磨色彩排序解谜游戏 MVP，完成遗留问题并新增游戏模式

## 关键工作

### 1. 理论最优步数计算（效率专家成就）
- **问题**：`handleWin` 传入 `theoreticalMin=0`，效率专家成就永远无法触发
- **方案**：重构 `solver.ts`，将 BFS 核心提取为 `solveBFS()` 返回 `{solvable, minSteps}`
- **实现**：
  - `getMinSteps()` 函数通过 BFS 最短路径计算理论最少步数
  - `Level` 类型新增 `minSteps` 字段
  - 关卡生成时对 ≤5 色关卡计算 minSteps（性能保护，复杂关卡跳过）
  - 游戏界面显示"💎 最优: N"金色徽章
  - 成就系统接入真实 minSteps 值

### 2. 限时挑战模式
- **目标**：增加快节奏游戏模式，提升留存和重复游玩价值
- **实现**：
  - 新增 `generateTimedLevel()` — 难度适中、多一个空管，追求快速通关
  - 120秒倒计时，实时显示剩余时间
  - 剩余 ≤10秒时计时器变红+脉冲动画
  - 时间到显示结算页（通过关数+分享+返回）
  - 限时最高分 localStorage 持久化
  - 3个限时成就：速度之星(5关)/风驰电掣(10关)/超越极限(20关)
  - 分享文案适配
