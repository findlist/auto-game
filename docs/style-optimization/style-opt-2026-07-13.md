## 具体修改内容

### 一、视觉层次与毛玻璃增强

#### 1. 游戏头部 sticky 毛玻璃（`.game-header`）
**优化前**：普通流式头部，滚动时与内容粘连，无层次感
**优化后**：
- `position: sticky; top: 0; z-index: 20` —— 滚动时头部固定可见，便于随时操作返回/提示按钮
- `backdrop-filter: blur(8px)` + `background: rgba(0, 0, 0, 0.08)` —— 毛玻璃透视效果，背景内容透过头部形成空间纵深
- `border-bottom: 1px solid rgba(255, 255, 255, 0.12)` —— 底部细分割线强化层次边界

#### 2. 关卡信息条毛玻璃（`.level-info-bar`）
- 添加 `backdrop-filter: blur(6px)` 和 `background: rgba(0, 0, 0, 0.12)`，与游戏头部风格统一

#### 3. 试管网格内嵌氛围光（`.tubes-grid`）
**优化前**：试管直接平铺在背景上，缺乏容器感
**优化后**：
- `background: radial-gradient(ellipse at center bottom, rgba(255, 255, 255, 0.08) 0%, transparent 70%)` —— 底部柔和光晕，营造"试管陈列台"的舞台感
- `border-radius: 12px` + padding 调整为 `20px 8px` —— 圆角容器包裹，视觉更聚焦

#### 4. 信息徽章内高光（`.level-badge, .difficulty-badge, .moves-badge`）
- 添加 `box-shadow: inset 0 1px 0 rgba(255, 255, 255, 0.3)` —— 顶部内高光让徽章有"玻璃按钮"质感
- hover 时 `transform: translateY(-1px)` —— 轻微上浮反馈

### 二、试管与球体视觉强化

#### 5. 试管过渡曲线优化（`.tube-container`）
**优化前**：`transition: all 0.2s ease` —— 线性回弹，略显生硬
**优化后**：`transition: transform 0.25s cubic-bezier(0.34, 1.3, 0.64, 1)` —— 弹性回弹曲线，hover 时有"被吸引"的轻微过冲感

#### 6. 试管 hover 边框响应（`.tube-container:hover .tube`）
- hover 时 `border-color: rgba(255, 255, 255, 0.6)` —— 鼠标悬停时试管玻璃边缘变亮，预示可交互

#### 7. 选中试管投影圆（`.tube-container.selected::after`）
**优化前**：选中状态仅有试管本体的发光边框
**优化后**：在试管下方新增投影圆（`radial-gradient` 椭圆 + `blur(4px)`），模拟"被托起"的光斑，强化选中态的空间感

### 三、设置与统计页交互增强

#### 8. 设置项左边框指示器（`.setting-item`）
**优化前**：设置项 hover 仅背景变化，反馈较弱
**优化后**：
- 默认 `border-left: 3px solid transparent`
- hover 时 `border-left-color: color-mix(in srgb, var(--primary) 60%, transparent)` —— 左侧出现主题色指示条，明确当前焦点项

#### 9. 设置开关弹性过渡（`.toggle-knob`）
- transition 改为 `cubic-bezier(0.34, 1.5, 0.64, 1)` —— 开关滑动时有更明显的弹性过冲
- 激活态添加 `box-shadow: 0 0 8px rgba(78, 205, 196, 0.5)` —— 开启时发出成功色光晕

#### 10. 统计卡片边框高亮（`.stat-card`）
- 默认 `border: 1px solid transparent`
- hover 时 `border-color: color-mix(in srgb, var(--primary) 40%, transparent)` —— 主题色边框淡入，配合原有阴影提升立体感

### 四、胜利与成就动效

#### 11. 胜利弹窗内容交错入场（`.win-card`、`.deadlock-card`）
**优化前**：弹窗内容整体一次性显现
**优化后**：通过 `.win-card > *:nth-child(n)` 配合 `animation-delay` 实现 0s/0.1s/0.2s/0.3s 的交错入场，标题→星级→按钮逐次出现，增强仪式感

#### 12. 成就卡片悬停光晕（`.achievement-card.unlocked:hover`）
**优化前**：已解锁成就 hover 仅轻微上浮
**优化后**：`box-shadow: var(--shadow), 0 0 0 1px rgba(78, 205, 196, 0.35), 0 0 16px rgba(78, 205, 196, 0.15)` —— 双层光晕（描边+外发光），让已解锁成就更醒目
- 注：曾考虑用 `::before` 添加左侧色彩条，因与现有 `border-left: 4px solid #4ECDC4` 冲突而放弃，改用强化 box-shadow 方案

#### 13. 进度条流光扫过（`.progress-bar-fill`）
**优化前**：进度条仅有静态渐变填充
**优化后**：新增 `::after` 伪元素 + `progress-shine` 动画，白色半透明光带以 2.5s 周期从左到右扫过进度条，让"进度"有持续流动感

### 五、移动端体验优化

#### 14. 触摸设备按钮反馈（`@media hover: none and pointer: coarse`）
**优化前**：移动端按钮的 hover 样式不触发，缺乏按下反馈
**优化后**：在 `pointer: coarse` 媒体查询下，为 `.btn`、`.btn-primary`、`.mode-grid-btn`、`.level-btn` 等 按钮 添加 `:active` 样式（`transform: scale(0.96)` + 背景加深），移动端按下时有明确的"按压"反馈

#### 15. 大屏桌面试管网格宽度限制
- 在宽屏媒体查询下为 `.tubes-grid` 设置 `max-width`，避免在超宽屏幕上试管被拉伸过散

### 六、FAB 导航与挑战横幅

#### 16. FAB 导航按钮悬停旋转（`.fab-nav`）
- hover 时 `transform: rotate(90deg)` —— 浮动导航按钮悬停时旋转，增加趣味性

#### 17. 每日/周挑战横幅图标动效
- 横幅图标 hover 时添加轻微弹跳，提升互动感

### 七、加载与占位

#### 18. 加载占位呼吸光晕（`.loading-placeholder`）
- 添加 `loading-breathe` 动画，opacity 在 0.5↔1.0 间以 1.8s 周期呼吸，避免加载时的"死板"等待感

### 八、死局预警动效

#### 19. 死局预警边框脉冲（`.deadlock-warning-tip` / `.deadlock-card`）
**优化前**：死局弹窗为静态显示，缺乏紧迫感
**优化后**：
- `animation: deadlock-warning-pulse 0.6s ease, deadlock-border-pulse 1.5s ease-in-out infinite 0.6s` —— 先一次性的警示脉冲，后持续的边框脉冲
- 边框颜色在 `rgba(255, 107, 107, 0.4)` ↔ `rgba(255, 107, 107, 0.8)` 间循环，红色脉冲暗示"需要采取行动"
- 注：曾考虑用 `::before` 插入 ⚠️ 图标，但检查发现 `.deadlock-warning-tip` 类名未在 GameBoard.tsx 中使用，且 padding-left 会破坏居中布局，故改用边框脉冲方案

### 九、滚动条与色块美化

#### 20. 模态弹窗滚动条美化
- 为 `.modal-content` 等可滚动区域自定义 `::-webkit-scrollbar` 样式（细窄、半透明、圆角），与整体玻璃风格统一

#### 21. 颜色选择器色块选中脉冲
- `.color-swatch.selected` 添加轻微脉冲动画，让选中状态更生动

### 十、无障碍支持

#### 22. prefers-reduced-motion 无障碍媒体查询
**优化前**：所有动画对所有用户强制播放，对前庭敏感用户不友好
**优化后**：在 `@media (prefers-reduced-motion: reduce)` 下：
- 将所有 `animation` 设为 `none`
- 将所有 `transition` 时长设为 `0.01ms`
- 保留必要的视觉反馈（如颜色变化），仅移除运动效果
- 尊重系统级的"减少动态效果"设置，提升无障碍体验
